## Trigger

Type `brainstorm` or `bs` to start design exploration.

## When to Use

- Before creating features
- Before building components
- Before adding functionality
- Before modifying behavior
- When comparing options

## Modes

| Mode | When | Time |
|------|------|------|
| Quick Think | Compare options, evaluate | 1-2 min |
| Light Design | Small feature, config change | 3-5 min |
| Full Design | New system, architecture | 15+ min |

## Quick Think Pattern

1. Gather context
2. Present structured analysis (table/comparison)
3. Give recommendation

## Light Design Pattern

1. Explore context
2. Ask 1-2 clarifying questions
3. Present design
4. Get approval

## Full Design Pattern

1. Explore project context
2. Ask clarifying questions
3. Propose 2-3 approaches
4. Present design sections
5. Write spec document
6. Review and refine

## Example Usage

```
brainstorm how to implement user authentication
bs should I use SQLite or PostgreSQL
brainstorm best approach for caching
```

## Output Types

- Quick Think: Analysis in chat
- Light Design: Design in chat
- Full Design: Spec saved to `docs/superpowers/specs/`
